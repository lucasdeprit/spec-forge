Login with Passkeys Allow users to sign in with passkeys while keeping the existing login flow available for password users, support teams and older devices. Reduce authentication friction for returning users, improve account security and preserve the current operational support path for password-based recovery. Users can sign in using passkeys. Users can still sign in using the existing login method. Users without a configured passkey are not blocked from accessing the application. Security-sensitive events are auditable so support can understand whether a passkey or fallback method was used. Users can recover access if a device with a registered passkey is lost. The existing login flow must not be removed. The change must stay within the authentication experience. Existing session cookies, refresh token behavior and rate limiting rules must remain compatible with mobile clients. Social login. Full login screen redesign. Enterprise SSO and organization-level identity provider configuration. Given a user with a configured passkey, when the user signs in, then authentication succeeds with the passkey. Given a user without a configured passkey, when the user signs in, then the existing login method remains available. The existing login flow continues to work after the change. Failed passkey attempts are logged with enough context for security review without storing sensitive credential material. draft
