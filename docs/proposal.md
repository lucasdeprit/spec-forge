# proposal.html

`proposal.html` is a global SpecForge artifact.

It describes what should exist.

A Proposal is intentionally implementation agnostic.

It should not mention:

- repositories
- files
- classes
- frameworks
- implementation strategy

Those details belong to Tasks and Implementations.

## Required Sections

- sf-proposal
- sf-title
- sf-description
- sf-goal
- sf-requirements
- sf-constraints
- sf-acceptance-criteria
- sf-status

## Optional Sections

- sf-out-of-scope
- sf-domains

## Status

Allowed statuses:

- draft
- ready
- completed
- archived

## Rules

- A Proposal can reference one or more workspace domains.
- A Proposal does not belong to a repository.
- A Proposal is the desired end state.
- A Proposal should be clear enough to generate Tasks.
